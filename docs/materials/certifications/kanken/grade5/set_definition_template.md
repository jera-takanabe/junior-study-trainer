# 漢検5級 セット定義テンプレート

## 0. 文書の目的

この文書は、漢検5級の問題セットを継続的に作成するためのセット定義テンプレートである。

問題データを作る前に、原則としてこのテンプレートに沿ってセット定義を作成する。

```text
1. セット定義
2. 問題データ作成
3. questions.js への統合
4. アプリで確認
5. 必要に応じて問題文を修正
```

---


## 1. 基本情報

```text
setId:
setName:
seriesName:
order:
version:
status:
```

記入例：

```text
setId: s003
setName: 基本熟語・日常語 03
seriesName: 基本熟語・日常語
order: 3
version: v0.1
status: draft
```

### status の意味

```text
draft      : 作成中
review     : 内容確認中
active     : アプリ投入済み
revised    : 修正版あり
archived   : 使用停止
```

---

## 2. セットの位置づけ

このセットが50セットロードマップ上のどこに属するかを書く。

```text
シリーズ：
番号帯：
位置づけ：
```

記入例：

```text
シリーズ：基本熟語・日常語シリーズ
番号帯：Set 001〜010
位置づけ：基本熟語・日常語シリーズの3本目
```

---

## 3. 目的

このセットで何を身につけるかを書く。

```text
・
・
・
```

目的は、単に「漢字を覚える」ではなく、以下のように具体化する。

```text
・日常生活や学校生活でよく使う熟語を読める
・文の中で意味を理解できる
・書き取りで使える
・同音異字や送り仮名で間違えやすい語を確認する
```

---

## 4. 対象漢字

このセットで主に扱う漢字を20字前後で定義する。

```text
・
・
・
```

書き方：

```text
漢字20字：
〇・〇・〇・〇・〇・〇・〇・〇・〇・〇
〇・〇・〇・〇・〇・〇・〇・〇・〇・〇
```

注意：

```text
・1セット20字前後を目安にする
・1026字の網羅確認は targets で行う
・対象漢字以外の漢字が熟語に入ることは許容する
・ただし、その場合は note や targets で意図を明確にする
```

## 6. 問題構成

問題数と内訳を定義する。

```text
総問題数：
移動用：
書き練習用：
```

詳細：

```text
読み：
選択・意味：
選択・対義語：
選択・類義語：
選択・同音異字：
書き取り：
送り仮名：
誤字訂正：
その他：
```

目安：

```text
1セット：40〜50問
移動用：20〜25問
書き練習用：15〜25問
```

## 7. category / level / questionType 方針

使用するカテゴリを定義する。

```text
category:
  - 漢検5級・移動用
  - 漢検5級・書き練習
```

使用する level を定義する。

```text
level:
  - 読み
  - 選択・意味
  - 選択・対義語
  - 選択・類義語
  - 選択・同音異字
  - 書き取り
  - 送り仮名
  - 誤字訂正
```

使用する questionType を定義する。

```text
questionType:
  - read
  - meaning
  - antonym
  - synonym
  - homophone
  - write
```

---

## 8. targets 方針

targets には、以下を入れる。

```text
1. 対象漢字1字
2. 熟語・語句
3. 同音異字や誤字訂正で比較対象になる語
```

例：

```javascript
targets: ["価", "値", "価値"]
```

同音異字の場合：

```javascript
targets: ["準", "備", "準備", "順備"]
```

注意：

```text
・1026字の網羅は漢字1字の targets で確認する
・苦手語句は熟語 targets で確認する
・誤字や混同語も targets に入れてよい
```

---

## 9. ID命名規則

正式IDは以下で統一する。

```text
kanken5-{setId}-{type}-{連番3桁}
```

例：

```text
kanken5-s003-read-001
kanken5-s003-write-001
kanken5-s003-meaning-001
```

注意：

```text
・一度使ったIDは再利用しない
・問題文を大きく変える場合は新しいIDにする
・進捗データはIDに紐づくため、ID変更は慎重に行う
```

---

## 10. 採用判定

アプリ投入前に以下を確認する。

```text
1. setId / setName / seriesName が全問に入っている
2. ID重複がない
3. 対象漢字が targets に含まれている
4. 移動用と書き練習用のバランスがよい
5. 問題文が曖昧でない
6. 答えが一意に決まる
7. note が必要十分である
8. index.html v0.6 のセット選択に表示される
9. 進捗CSVに set_id / set_name が出る
10. バックアップJSONに sets が含まれる
```
