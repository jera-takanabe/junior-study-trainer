# Science Local Prototype Policy v0.1

## 1. 本資料の目的

本資料は、理科の定期テスト対策プロトタイプにおいて、学校副教材をもとに復元・再構成した問題データおよび図素材を、Junior Study Trainer でどのように扱うかを整理するものである。

`question_data_architecture_v0_7.md` では、最初の学校教科プロトタイプとして理科を候補とし、副教材ベースの80点対策問題と、教科書ベースの100点対策問題を分けて検証する方針を定めた。

今回、実際の定期テスト範囲の副教材PDFと解説・解答PDFをもとに、問題再現版および一問一答データを試作した結果、公開可能な自作問題データと、副教材由来のローカル確認用データを明確に分けて扱う必要があることが分かった。

## 2. 今回作成した理科プロトタイプの位置づけ

今回作成した `Set001` は、次の内容を対象とする。

| 項目 | 内容 |
|---|---|
| 教科 | 理科 |
| 学年 | 中学1年 |
| テスト | 1学期中間テスト |
| 目的 | 副教材から出題される80点分への対応 |
| 対象範囲 | 花のつくり・花の変化・マツのなかま |
| 問題数 | 拡張版20問、現行画面確認版14問 |
| 特徴 | 図付き問題を含む |
| 公開区分 | ローカル確認専用 |

この `Set001` は、元副教材の設問や図をもとに復元・再構成している。そのため、公開アプリへ投入する問題データではなく、家庭内での学習およびアプリ構造検証のためのローカル専用データとして扱う。

## 3. データの区分

理科問題データは、当面、次の2種類に分けて管理する。

### 3.1 ローカル専用データ

学校副教材や教科書をもとに、元問題の再現性を重視して作成したデータである。

対象例:

- 副教材の問題を解き直せるように復元した問題文
- 副教材の図を切り出した画像
- 解答冊子に対応づけた解答・解説
- 副教材の問題を一問一答化したデータ

この区分のデータは、家庭内での定期テスト対策、ローカル環境での画面表示確認、問題形式やデータ構造の検証に限定して使用する。

この区分のデータは、公開リポジトリおよび GitHub Pages に配置しない。

### 3.2 公開可能な自作データ

副教材や教科書の構成を参考にしつつも、問題文、解説、図素材を自作したデータである。

対象例:

- 用語を確認するために新たに作成した一問一答
- 自作の模式図を用いた確認問題
- 一般的な学習内容をもとに新規作成した理由説明問題

公開アプリへ理科を追加する場合は、原則としてこの区分のデータのみを `app/` 配下へ配置する。

## 4. ローカル専用データの配置方針

副教材由来のローカル専用データは、リポジトリの作業フォルダ内に置く場合でも、Git管理対象外となる `.local/` 配下に配置する。

想定構成は次のとおりである。

```text
junior-study-trainer/
├─ app/
│  ├─ index.html
│  ├─ questions.js
│  └─ ...
├─ data/
├─ docs/
└─ .local/
   └─ rika_app_data_prototype_s001_local_only/
      ├─ app_trial/
      │  └─ questions_science_s001_text_trial_local_only.js
      ├─ data/
      │  └─ school/
      │     └─ science/
      │        └─ jh1/
      │           └─ term1_midterm/
      │              └─ workbook/
      │                 └─ sets/
      │                    ├─ s001_extended_local_only.js
      │                    └─ s001_extended_local_only.json
      ├─ local_assets/
      │  └─ figures/
      └─ docs/
```

`.local/` は、教材原本・復元データ・切り出し画像・ローカル確認用データなど、公開しない素材を一時的または継続的に保管する場所として扱う。

## 5. Git管理対象外とする方法

`.local/` は、公開リポジトリへコミットしない。

個人のローカル環境だけで除外するため、本プロジェクトでは、まず `.git/info/exclude` に次の設定を追加する方式を採用する。

```text
.local/
```

この方式であれば、公開リポジトリ側にローカル教材用フォルダの運用設定を追加せず、各作業環境で副教材由来データの誤コミットを防止できる。

ただし、今後、複数環境で同様のローカル専用教材を扱う運用になった場合は、`.gitignore` に明示する方式へ変更することを検討する。

## 6. 現行アプリでの動作確認方針

現行アプリは、`app/questions.js` を読み込んで動作する。また、現時点では `materials` による図画像表示には対応していない。

そのため、理科 `Set001` の確認は次の2段階で行う。

### 6.1 第1段階：テキスト問題の表示確認

図を必要としない14問のみを含む確認用ファイルを用いて、既存画面で理科問題が動作するかを確認する。

確認対象:

- セット名が表示されること
- 問題文と解答が表示されること
- 出題タイプの分類が表示できること
- 正解、不正解、パスの記録が動作すること
- 既存の漢検用画面で理科問題の文章量を扱えること

この確認では、ローカル作業中に限り、`app/questions.js` を確認用理科データへ一時的に差し替える。

### 6.2 第2段階：図付き問題の表示確認

第1段階の確認後、`materials` の画像を1枚表示できる最小限の変更を検討する。

確認対象:

- 問題データに `materials` を持たせられること
- 画像付き問題を画面に表示できること
- 図を見て答える問題でも、既存の正誤記録を利用できること
- 画像参照パスをローカル専用運用で無理なく扱えること

この段階では、汎用的な複数資料ビューアや本格的な教材管理機能までは作成しない。

## 7. 問題データ項目の扱い

理科プロトタイプでは、既存の問題データとの互換性を保ちながら、次の項目を追加検証する。

| 項目 | 役割 | 例 |
|---|---|---|
| `sourceType` | 出典種別 | `workbook` |
| `difficultyLevel` | 到達目標区分 | `basic_80` |
| `questionType` | 問題形式 | `short_answer`, `image_short_answer`, `written_reason` |
| `materials` | 図表などの補助資料 | 画像参照情報 |
| `displayFormat` | 表示形式 | `text`, `text_image` |
| `sourceRef` | 出典確認用情報 | 本誌ページ・設問番号 |

現行アプリの `level` は画面上の出題タイプ表示と関連しているため、80点対策／100点対策の区分には使用しない。到達目標の区分には `difficultyLevel` を使用する。

## 8. 公開版へ移行する場合の条件

将来、理科問題を公開アプリへ追加する場合は、ローカル専用データをそのまま移すのではなく、次の条件を満たす公開可能版を新たに作成する。

- 問題文を自作または十分に再構成する
- 図素材を自作または利用可能な素材に置き換える
- 副教材PDFや切り出し画像を含めない
- 公開用データとローカル専用データを混在させない
- 公開前に `app/` 配下へ含まれるファイルを確認する

## 9. 当面の進め方

理科プロトタイプは、次の順で進める。

1. `.local/` 配下に `Set001` のローカル専用データを配置する。
2. テキスト問題14問を用いて、現行アプリとの互換性を確認する。
3. 確認結果を記録する。
4. `materials` の画像を1枚表示する最小変更を検討する。
5. 図付き問題6問を含む20問版で表示確認する。
6. データ構造の妥当性を判断してから、次セットの問題データ化へ進む。

## 10. 暫定結論

副教材由来の定期テスト対策データは、学習効果の観点では元問題に近い形で反復できることが重要である。一方で、公開アプリへそのまま配置するデータとは明確に分ける必要がある。

そのため、理科 `Set001` は `.local/` に置くローカル専用プロトタイプとして扱い、現行アプリへの組み込み確認と、図付き問題データ構造の検証に用いる。

公開アプリへ理科を追加する場合は、本プロトタイプで得られた設計上の知見を利用しつつ、公開可能な自作問題データを別途作成する。
