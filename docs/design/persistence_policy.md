# タイムアタック一問一答 v0.5 永続化方針

## 1. 問題意識

複数日にわたって試験運用するには、PCを落としても学習結果が残る必要がある。

現状の v0.4 は `localStorage` を使っているため、本来は同じブラウザ・同じURLで開けば保存される想定だが、以下の場合はリセットされたように見える。

```text
・index.html の保存場所を変えた
・ファイル名やフォルダを変えた
・別ブラウザで開いた
・ブラウザが終了時にサイトデータを削除している
・シークレットモードで開いている
・v0.3 と v0.4 で保存キーが違う
・ブラウザやセキュリティ設定により file:// の保存が不安定
```

したがって、複数日運用の前に、永続化を強化する。

---

## 2. 方針

v0.5 では、AWS化の前に、ローカル運用で進捗を守る仕組みを追加する。

```text
1. localStorage 保存は継続する
2. 画面上に保存状態を表示する
3. 進捗・履歴をJSONでバックアップできるようにする
4. JSONから復元できるようにする
5. CSVは分析用、JSONは復元用として分ける
```

---

## 3. CSVとJSONの役割分担

### 3.1 CSV

CSVは、人間がExcelなどで確認するためのもの。

```text
・履歴確認
・進捗分析
・弱点確認
・バックアップの参考
```

ただし、CSVから完全復元するのは難しい。

### 3.2 JSON

JSONは、アプリに戻すためのバックアップ。

```text
・進捗復元
・履歴復元
・PC変更時の移行
・ブラウザデータ消失時の復旧
```

---

## 4. v0.5で追加する画面ボタン

設定画面に以下を追加する。

```text
保存確認
バックアップJSON出力
バックアップJSON読込
```

既存のボタンは残す。

```text
履歴CSV
進捗CSV
履歴削除
進捗リセット
```

---

## 5. バックアップJSONの形式

```javascript
{
  appName: "timed-quiz-trainer",
  version: "v0.5",
  exportedAt: "2026-05-26T00:00:00.000Z",
  historyKey: "timedQuizTrainerHistoryV04",
  progressKey: "timedQuizTrainerProgressV04",
  history: [...],
  progress: {...}
}
```

---

## 6. 運用ルール

複数日テスト中は、以下を推奨する。

```text
1日の学習後：バックアップJSON出力
週1回：進捗CSV出力
問題データ差し替え前：バックアップJSON出力
進捗リセット前：バックアップJSON出力
```

---

## 7. PCを落としても残るかの確認手順

```text
1. v0.5で5問実施
2. 設定画面に戻る
3. 未着手・着手済み・復習ポイントが変わることを確認
4. ブラウザを閉じる
5. PCを再起動する
6. 同じ index.html を同じブラウザで開く
7. 進捗が残っているか確認
8. 残っていなければ、バックアップJSONから復元する
```

---

## 8. AWS化との関係

将来的にAWS化する場合、JSONバックアップの構造はそのままクラウド保存のデータモデルに近い。

```text
localStorage
↓
JSONバックアップ
↓
DynamoDB保存
```

そのため、v0.5の永続化強化はAWS化前の無駄な作業ではなく、クラウド化の前段階として有効。
