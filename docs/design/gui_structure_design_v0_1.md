# GUI構造・画面導線設計 v0.1

## 1. この文書の目的

この文書は、Junior Study Trainer の今後のGUI構造と画面導線を整理するための設計メモである。

目的は、画面の見た目を細かく決めることではなく、次に作成する「問題データ共通メタデータ設計」が、実際の画面・操作・進捗管理とずれないようにすることである。

特に、これまで整理した次の設計との整合を確認する。

```text
関連設計
├─ 学校教科カタログ
├─ 検定カタログ
├─ カタログ横断と問題データ設計
├─ 目的別問題集設計
├─ 印刷用問題出力設計
└─ 図・表・グラフつき問題データ設計
```

## 2. 現在のGUI前提

現在のアプリは、教材を選んでタイマー付き一問一答を行う構造である。

```text
現在の主な画面・機能
├─ 教材選択
├─ 問題表示
├─ タイマー
├─ 解答表示
├─ 手動正誤
├─ 履歴
├─ 進捗
└─ バックアップ
```

これは、現行の「教材単位で学習する」使い方には合っている。

一方で、今後は次の使い方が増える。

```text
今後増える使い方
├─ 教材から選ぶ
├─ 目的から選ぶ
├─ 検定から選ぶ
├─ 弱点から選ぶ
├─ 印刷する
├─ 図表つき問題を解く
├─ 英語音声を聞く
└─ スピーキング練習をする
```

## 3. 基本方針

```text
基本方針
├─ 現行の教材選択UIは残す
├─ 新たに「目的から選ぶ」導線を追加する
├─ 学校教科と検定を同じUIの中で扱う
├─ 画面は増やしすぎず、入口を整理する
├─ 目的別問題集は、教材セットとは別の選択肢として扱う
├─ 印刷は学習開始とは別の出力導線にする
├─ 図表・音声・スピーキングは将来拡張枠として置く
├─ 子どもが使う画面は単純にする
└─ 親が管理する画面は詳細でもよい
```

将来的なトップ画面は、次の2系統を基本にする。

```text
トップ

│  ├─ 教材から選ぶ

│  └─ 前回の続き
│
├─ 印刷する

│  └─ 総合パック
│
```

## 5. 子ども用と親用の分離

画面は、利用者によって必要な情報量が違う。

```text
子ども用
├─ 今日やるものが分かる
├─ ボタンが少ない
├─ すぐ開始できる
├─ 結果が分かる
└─ 難しい設定は見せない

親用
├─ 教材を選ぶ
├─ 目的別問題集を作る
├─ 出題条件を調整する
├─ 進捗を見る
├─ 弱点を見る
├─ 印刷する
└─ バックアップする
```

初期実装では厳密なユーザー切替は不要だが、設計上は意識する。

## 6. 学習開始までの導線

### 6.1 教材から選ぶ

現在の方式を維持する。

```text
教材から選ぶ
├─ 理科教科書
├─ 地理
├─ 歴史
├─ 国語定期テスト
├─ 英語定期テスト
├─ 数学定期テスト
└─ 今後追加する教材
```

向いている用途：

```text
向いている用途
├─ 学校教材を順に学ぶ
├─ 既存問題セットを使う
├─ 定期テスト範囲を確認する
└─ 現行アプリを壊さず運用する
```

### 6.2 目的から選ぶ

目的別問題集の入口。

```text
目的から選ぶ
├─ 定期テスト対策
├─ 検定対策
├─ 弱点補強
├─ 先取り
├─ 復習
├─ 本番形式演習
├─ スピンバイク40問
└─ 印刷用演習
```

向いている用途：

```text
向いている用途
├─ 何をやるべきか迷わせない
├─ 検定日から逆算する
├─ 弱点を集中して潰す
├─ 短時間学習をする
└─ 横断的に問題を集める
```

### 6.3 今日のおすすめ

将来的な自動提案枠。

```text
今日のおすすめ
├─ 直近の検定
├─ 定期テスト予定
├─ 最近の誤答
├─ しばらく触っていない問題
├─ 学習計画
└─ 親が指定した優先課題
```

初期実装では行わない。

### 6.4 前回の続き

現在の履歴・進捗と相性がよい。

```text
前回の続き
├─ 前回の教材
├─ 前回の目的別問題集
├─ 未完了の問題
└─ 中断した本番形式演習
```

## 7. 検定対策の見せ方

検定は「検定 × 級」で表示する。

```text
検定対策
├─ 数検
│  ├─ 6級
│  └─ 5級
├─ 英検
│  └─ 3級
└─ 漢検
   └─ 4級
```

各級の中は、目的別問題集で分ける。

```text
数検5級
├─ 1次 計算技能
├─ 2次 数理技能
├─ 正負の数
├─ 文字式
├─ 一次方程式
├─ 図形
├─ 関数
└─ 本番形式

英検3級
├─ 語彙
├─ Reading
├─ Writing
├─ Listening
├─ Speaking
└─ 本番形式

漢検4級
├─ 読み
├─ 書取
├─ 部首
├─ 送り仮名
├─ 四字熟語
└─ 総合演習
```

## 8. 定期テスト対策の見せ方

定期テストは、学校教科と回次で表示する。

```text
定期テスト対策
├─ 第1回
│  ├─ 国語
│  ├─ 数学
│  ├─ 英語
│  ├─ 理科
│  └─ 社会
├─ 第2回
└─ 第3回
```

各教科の中では、明示ID方式の固定セットを優先する。

```text
数学 第1回
├─ 全範囲
├─ 計算だけ
├─ 文章題だけ
├─ 間違えた問題
└─ 直前確認
```

## 9. 弱点補強の見せ方

弱点補強は、学習履歴から自動抽出する将来機能である。

```text
弱点補強
├─ 最近間違えた問題
├─ 2回以上間違えた問題
├─ 正答率が低い問題
├─ しばらく解いていない問題
├─ タグ別の弱点
└─ 親が指定した弱点
```

初期段階では、明示ID方式で「間違えた問題セット」を作るだけでもよい。

## 10. スピンバイク・車移動用の見せ方

短時間・口頭回答向けの専用入口を用意する。

```text
スピンバイク40問
├─ 英単語
├─ 漢字の読み
├─ 社会一問一答
├─ 理科用語
├─ 暗算
├─ 公式確認
└─ ランダムミックス
```

出題条件：

```text
条件
├─ 書かなくてよい
├─ 図表を使わない
├─ 長文を使わない
├─ 1問あたり短時間
├─ ランダム出題
└─ 40問程度
```

## 11. 印刷導線

印刷は、学習開始とは別の導線にする。

```text
印刷する
├─ 目的別問題集から印刷
├─ 教材から印刷
├─ 弱点だけ印刷
├─ 本番形式で印刷
└─ 解答・解説も印刷
```

印刷モード：

```text
印刷モード
├─ 問題用紙のみ
├─ 解答用紙
├─ 解答一覧
├─ 解答・解説
└─ フルパック
```

理由：

```text
理由
├─ 画面で解く問題と紙で解く問題は目的が違う
├─ 書取・作図・英作文は紙に向く
├─ 親が印刷して渡す運用が多い
└─ 学校提出用にも使う
```

## 12. 図表つき問題の画面導線

図表つき問題は、通常問題と同じ流れで扱うが、表示に追加操作が必要になる。

```text
図表つき問題
├─ 図を表示
├─ 拡大表示
├─ 表を表示
├─ グラフを表示
├─ 地図を表示
├─ 年表を表示
└─ 印刷向けに大きく出力
```

画面上では、まずシンプルに「図を表示する」だけでよい。

```text
初期対応
├─ 画像1枚
├─ 問題文の前後に表示
├─ 拡大は後回し
└─ 印刷はMarkdown出力で対応
```

## 18. GUIから見たメタデータ要件

GUI設計から見ると、問題データ共通メタデータには次が必要になる。

```text
GUIで必要なメタデータ
├─ 表示名
├─ 教材ID
├─ 目的別問題集ID
├─ 教科
├─ 検定種別
├─ 級
├─ 学習領域
├─ 問題形式
├─ 難度
├─ 書く必要があるか
├─ 図表があるか
├─ 音声があるか
├─ スピーキング対象か
├─ 印刷可能か
├─ 所要時間目安
└─ 進捗管理単位
```

この確認により、次の「問題データ共通メタデータ設計」で扱うべき項目が明確になる。

## 19. 実装順序の推奨

```text
段階1：
  現行GUI維持
  設計文書のみ追加

段階2：
  目的から選ぶ入口を追加
  まずは明示ID方式の目的別問題集のみ

段階3：
  印刷用Markdown出力をアプリ外スクリプトで試す

段階4：
  図表つき問題の画像1枚表示を試す

段階5：
  抽出条件方式の目的別問題集を試す

段階6：
  進捗を目的別問題集単位へ拡張

段階7：
  音声再生・スピーキング評価へ進む
```

## 22. まとめ

GUI設計上の中心は、次の二本立てである。

```text
教材から選ぶ：
  現行アプリの安定運用

目的から選ぶ：
  検定対策・弱点補強・先取り・印刷・本番形式への拡張
```

問題データ共通メタデータは、この画面導線を支えるために設計する。

当面は、GUIを大きく変えず、設計と新規問題作成時のメタデータ意識から始める。
