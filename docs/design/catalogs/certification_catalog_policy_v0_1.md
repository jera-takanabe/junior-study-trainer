# 検定カタログ共通方針 v0.1

## 1. この文書の目的

この文書は、Junior Study Trainer における検定カタログの共通方針を整理するための設計メモである。

学校教科カタログは「教科 × 学年」を基本単位とする。一方、検定カタログは「検定 × 級」を基本単位とする。

ただし、検定は上位級に進むにつれて出題範囲・出題形式・必要技能が広がる。そのため、次回受験する級からスモールスタートしつつ、上位級へ拡張できる構造にする。

## 2. 正本と補助資料

検定カタログの正本は、公式サイト・公式要項の出題範囲とする。

優先順位は次の通り。

```text
第1根拠：公式サイト・公式要項の出題範囲
第2根拠：公式の過去問・サンプル問題・問題形式
第3根拠：公式の級の目安・到達目標・審査基準
第4根拠：市販教材・学校教材・学習アプリ
```

公式サイト本文の長文引用は避ける。リポジトリには、URL、参照日、要約、設計上の分類を記録する。

## 3. 作成単位

### 3.1 学校教科

学校教科は学年ごとに作成する。

```text
学校教科：
  教科 × 学年

例：
  中1数学
  中1英語
  中1理科
  中1地理
  中1歴史
```

### 3.2 検定

検定は、次回受験する級を単位として作成する。

```text
検定：
  検定 × 級

例：
  英検4級
  漢検4級
  数検6級
```

ただし、設計上は検定全体を見据えた共通軸を持たせる。

## 4. 検定カタログの基本構造

```text
検定全体
└─ 検定種別
   └─ 級
      ├─ 出題範囲
      ├─ 出題形式
      ├─ 必要技能
      ├─ 知識項目
      ├─ 問題タイプ
      ├─ 下位級からの継承
      ├─ この級で追加される内容
      └─ 上位級への接続
```

## 5. 上位級へ拡張するための設計

級ごとのカタログには、次の観点を持たせる。

```text
prerequisiteFromLowerGrades:
  下位級から引き継ぐ内容

newAtThisGrade:
  この級で新しく増える内容

emphasizedAtThisGrade:
  この級で特に重要になる内容

nextGradePreparation:
  次の級につながる内容
```

これにより、最初は次回受験級だけを作成しても、上位級へ進んだときに作り直しにならないようにする。

## 7. 問題データで想定する共通属性

将来の問題データでは、少なくとも次の属性を持てるようにする。

```text
certification:
  検定種別
  例）eiken / kanken / suken

grade:
  級
  例）4級 / 6級

domainTags:
  領域
  例）reading / listening / writing / speaking / 漢字 / 図形

questionTypeTags:
  出題形式
  例）空所補充 / 並べ替え / 読み / 書き / 計算 / 文章題

knowledgeTags:
  知識項目
  例）過去形 / 不規則動詞 / 部首 / 比 / 面積

skillTags:
  必要技能
  例）聞き取る / 読み取る / 書く / 説明する / 計算する

source:
  参照元
  例）公式サイト / 公式過去問 / 市販教材 / 自作
```

## 8. 音声・発話機能を見据えた設計

英語学習・英検対策では、将来的に音声を扱えるようにする。

### 8.1 リスニング・ヒアリング用の音声再生機能

将来機能として、問題データに音声素材を紐づけられるようにする。

用途：

```text
用途
├─ 英検リスニング対策
├─ 英語教科書本文の音読確認
├─ 単語・例文の音声再生
├─ 会話文の聞き取り
└─ 移動中・スピンバイク中の聞き流し学習
```

想定属性：

```text
audio:
  enabled: true / false
  type: word / sentence / dialogue / passage / instruction
  source: official / textbook / generated / recorded
  fileRef: ローカルまたは将来の保存先
  transcriptRef: 文字起こし
  playbackMode: once / repeat / shadowing
```

この機能は急がない。英語学習用素材として使えるよう、データ設計上の余地を残す。

### 8.2 スピーキング用の音声インプット＋AI評価

将来機能として、学習者の発話を録音し、AIで評価できるようにする。

用途：

```text
用途
├─ 英検スピーキング対策
├─ 教科書本文の音読
├─ 基本文の発話練習
├─ 自己紹介・スピーチ練習
├─ 発音・流暢さ・内容の確認
└─ 面接練習
```

想定属性：

```text
speaking:
  enabled: true / false
  promptType: read_aloud / answer_question / speech / interview
  expectedText: 模範文または参考文
  evaluationCriteria:
    - pronunciation
    - fluency
    - accuracy
    - content
    - grammar
    - vocabulary
  aiFeedback: 将来利用
```

この機能も急がない。アプリ開発がある程度落ち着いた後に検討する。

## 9. 印刷用問題一覧・解答一覧

PCやスマホを使えない環境、紙での演習、学校宿題提出に対応するため、印刷用出力を設計に含める。

### 9.1 目的

```text
目的
├─ PCが使えない環境で印刷して学習する
├─ 学校宿題提出用に紙で解く
├─ 定期テスト前に紙で総復習する
├─ 書き問題・記述問題を紙で練習する
├─ 保護者が採点しやすい形にする
└─ アプリで扱いにくい図・グラフ・表を紙で扱う
```

### 9.2 出力単位

```text
印刷出力
├─ 問題一覧
├─ 解答一覧
├─ 解答・解説
├─ チェック欄付き問題
├─ 書き込み用スペース付き問題
└─ 図・グラフ・表つき問題
```

### 9.3 実装タイミング

印刷用出力は、図・グラフ・表を問題に入れる設計と関係が深い。

優先度は次の通り。

```text
優先度
├─ 先行検討可
│  └─ テキスト中心の問題一覧・解答一覧
├─ 図表対応と同時期
│  └─ 図・グラフ・表つき問題の印刷
└─ 後続拡張
   └─ レイアウト調整、PDF出力、宿題提出用フォーマット
```

現時点では実装しないが、問題データ設計時点で紙出力を妨げない構造にする。

## 10. 既存アプリ設計との関係

現時点のアプリは、タイマー付き一問一答を中心とする。

今後の拡張は次の順序を想定する。

```text
現行：
  タイマー付き一問一答
  手動正誤
  教材選択
  進捗・履歴

近い将来：
  目的別問題集
  出題条件の整理
  問題データ属性の拡張
  印刷用問題一覧・解答一覧

中期：
  図・グラフ・表つき問題
  紙出力レイアウト
  書き問題・記述問題対応

後期：
  音声再生
  音声入力
  AIによるスピーキング評価
```

## 12. 実装時の注意

この文書は設計方針であり、現時点では次を行わない。

```text
現時点で行わないこと
├─ app/data/*.js の変更
├─ manifest への追加
├─ 音声ファイルの追加
├─ 音声録音機能の実装
├─ AI評価機能の実装
├─ PDF出力機能の実装
└─ 印刷レイアウトの確定
```

ただし、今後のデータ設計で、音声・発話・印刷出力を妨げないようにする。
