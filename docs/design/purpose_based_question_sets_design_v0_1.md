# 目的別問題集設計 v0.1

## 1. この文書の目的

この文書は、Junior Study Trainer における「目的別問題集」の設計方針を整理するための設計メモである。

ここでいう目的別問題集とは、単に教材ファイルを選ぶのではなく、学習目的に応じて問題を束ねたものである。

例：

```text
目的別問題集
├─ 定期テスト対策
├─ 検定対策
├─ 弱点補強
├─ 先取り
├─ 復習
├─ 本番形式演習
├─ 印刷用演習
└─ 車移動・スピンバイク用の短時間演習
```

この文書では、すぐにアプリを改修するのではなく、今後の問題データ・進捗管理・印刷出力へつなげるための設計を整理する。

関連する設計文書は次の通り。

```text
関連設計
├─ id_and_code_policy_v0_1.md
├─ question_metadata_schema_design_v0_1.md
├─ metadata_migration_plan_v0_1.md
└─ printable_question_output_design_v0_1.md
```

## 2. 現在の前提

現在のアプリは、教材セット単位で問題を選ぶ。

```text
現在の単位
├─ questionSetId
├─ 教材名
├─ app/data/*.js
├─ manifest
├─ 教材単位の進捗
└─ 教材単位の履歴
```

これは現在の学習には有効である。

一方で、今後は同じ問題を複数の目的で使いたい。

例：

```text
同じ問題の利用例
├─ 中1数学の正負の数
│  ├─ 学校定期テスト対策
│  ├─ 数検5級対策
│  ├─ 弱点補強
│  └─ 先取り復習
│
├─ 中1英語の文法
│  ├─ 学校定期テスト対策
│  ├─ 英検3級対策
│  ├─ ライティング準備
│  └─ スピーキング準備
│
└─ 漢字問題
   ├─ 学校国語
   ├─ 漢検4級
   ├─ 語彙強化
   └─ 書き取り練習
```

そのため、教材セットとは別に、目的別問題集を設計する。

## 3. 基本方針

```text
基本方針
├─ 目的別問題集は、教材セットとは別の概念とする
├─ 1つの問題は複数の目的別問題集に属してよい
├─ 明示ID方式と抽出条件方式の両方を使えるようにする
├─ 定期テスト対策は明示ID方式を優先する
├─ 検定対策・弱点補強は抽出条件方式を優先する
├─ 本番形式演習は固定セットとして扱えるようにする
├─ 印刷用出力を前提にした問題集も作れるようにする
└─ 既存アプリの動作は壊さず、段階的に導入する
```

## 5. 目的別問題集の分類

### 5.1 定期テスト対策

```text
定期テスト対策
├─ 学校配布範囲に合わせる
├─ 教科書・ワーク・プリントに対応する
├─ 問題順を固定しやすい
├─ 試験直前に使う
└─ 明示ID方式を優先する
```

理由：

```text
定期テストは、
├─ 出題範囲が限定される
├─ 学校独自の重点がある
├─ 問題順や構成が重要になる
└─ 余計な範囲を混ぜたくない
```

### 5.2 検定対策

```text
検定対策
├─ 公式範囲に合わせる
├─ 級ごとの出題形式に合わせる
├─ 下位級からの継承を含む
├─ 上位級への接続を意識する
└─ 抽出条件方式を優先する
```

例：

```text
数検5級
├─ 正負の数
├─ 文字式
├─ 一次方程式
├─ 作図
├─ 空間図形
├─ 比例・反比例
└─ 度数分布

英検3級
├─ 語彙
├─ 短文空所補充
├─ 長文読解
├─ Eメール返信
├─ 意見英作文
├─ リスニング
└─ スピーキング

漢検4級
├─ 読み
├─ 書取
├─ 部首
├─ 送り仮名
├─ 同音・同訓異字
├─ 四字熟語
└─ 熟語の構成
```

### 5.3 弱点補強

```text
弱点補強
├─ 誤答履歴から作る
├─ 苦手タグから作る
├─ 正答率の低い問題を集める
├─ 期間を区切って復習する
└─ 抽出条件方式を優先する
```

例：

```text
弱点補強の条件例
├─ 正答率70%未満
├─ 直近3回で2回以上ミス
├─ mistakeCount >= 2
├─ tag = 分数
├─ tag = 英検3級語彙
└─ tag = 漢検4級書取
```

### 5.4 先取り

```text
先取り
├─ 学校進度より先の内容
├─ 検定受験に必要な内容
├─ 既習範囲との接続を明示する
├─ 難度を段階的に上げる
└─ カタログ参照を重視する
```

例：

```text
先取り
├─ 数検5級の中1数学
├─ 英検3級の中学卒業程度英語
├─ 漢検4級の中学校在学程度漢字
└─ 歴史年表の広い流れ
```

### 5.5 復習

```text
復習
├─ 過去に学習済みの問題
├─ 期間をあけて再出題
├─ 正答済みでも忘却対策として出す
├─ 簡単な確認問題を含む
└─ 抽出条件方式を優先する
```

例：

```text
復習条件
├─ lastAnsweredAt が7日以上前
├─ lastAnsweredAt が30日以上前
├─ masteryLevel が低い
├─ correctStreak が途切れた
└─ 重要タグを含む
```

### 5.6 本番形式演習

```text
本番形式演習
├─ 問題数を固定する
├─ 時間を固定する
├─ 出題形式を本番に近づける
├─ 採点基準を明確にする
└─ 明示ID方式または固定条件方式を使う
```

例：

```text
本番形式
├─ 数検6級 30問
├─ 数検5級 1次30問
├─ 数検5級 2次20問
├─ 英検3級 一次形式
├─ 英検3級 二次面接形式
└─ 漢検4級 総合形式
```

### 5.7 印刷用演習

```text
印刷用演習
├─ 書く必要がある問題
├─ 途中式が必要な問題
├─ 作図が必要な問題
├─ 英作文が必要な問題
├─ 書取が必要な問題
└─ 答案用紙が必要な問題
```

印刷用演習は、アプリ上の一問一答とは別に、紙に出すことを前提にする。

## 6. 明示ID方式

明示ID方式は、問題IDを直接並べて問題集を作る方式である。

### 6.1 形式イメージ

```text
purposeSetId:
  purpose_math_test1_final_review_001

selectionMode:
  explicitIds

questions:
  - questionSetId: "school_math_jhs1_test1"
    questionId: "math_test1_m001_q001"

  - questionSetId: "school_math_jhs1_test1"
    questionId: "math_test1_m001_q002"

  - questionSetId: "school_math_jhs1_test1"
    questionId: "math_test1_m003_q014"

  - questionSetId: "school_math_jhs1_test1"
    questionId: "math_test1_m013_q020"
```

明示ID方式では、questionId 単独ではなく、questionSetId と questionId の組で問題を参照する。

```text
理由
├─ questionId の衝突に備える
├─ 教材セットをまたぐ目的別問題集に対応する
├─ sourceFile を manifest から特定しやすくする
└─ ID 文字列の意味に依存しない実装にする
```

### 6.2 向いている用途

```text
向いている用途
├─ 定期テスト対策
├─ 学校配布プリント対応
├─ 直前確認
├─ 本番形式演習
├─ 親が厳選した問題
└─ 間違えた問題だけの固定再テスト
```

### 6.3 メリット

```text
メリット
├─ 意図どおりの問題集を作れる
├─ 問題順を固定できる
├─ 余計な問題が混ざらない
├─ テスト直前に安心して使える
└─ 紙出力と相性がよい
```

### 6.4 デメリット

```text
デメリット
├─ 問題追加時に手動更新が必要
├─ タグ活用の恩恵が少ない
├─ 大量問題では管理が重い
└─ 条件変更に弱い
```

## 7. 抽出条件方式

抽出条件方式は、タグやメタデータを使って問題を抽出する方式である。

### 7.1 形式イメージ

```text
purposeSetId:
  suken_grade5_signed_numbers

selectionMode:
  query

conditions:
  certification:
    - suken
  grade:
    - 5級
  domainTags:
    - 数と式
  knowledgeTags:
    - 正の数
    - 負の数
  questionTypeTags:
    - 計算
```

### 7.2 向いている用途

```text
向いている用途
├─ 検定対策
├─ 弱点補強
├─ 復習
├─ 先取り
├─ ランダム演習
├─ 車移動・スピンバイク演習
└─ 問題追加に追随する問題集
```

### 7.3 メリット

```text
メリット
├─ 問題追加に強い
├─ 条件を変えるだけで問題集を作れる
├─ 弱点補強と相性がよい
├─ 検定カタログと相性がよい
└─ 横断的な問題集を作りやすい
```

### 7.4 デメリット

```text
デメリット
├─ タグ設計が甘いと意図しない問題が混ざる
├─ 問題順の制御が難しい
├─ 出題範囲の境界があいまいになる
└─ 初期設計がやや重い
```

## 8. 混合方式

混合方式は、明示ID方式と抽出条件方式を組み合わせる。

### 8.1 形式イメージ

```text
purposeSetId:
  eiken_grade3_final_weakness_review

selectionMode:
  mixed

explicitQuestionIds:
  - eiken_g3_vocab_Q001
  - eiken_g3_writing_Q010

conditions:
  certification:
    - eiken
  grade:
    - 3級
  weakOnly:
    true
  mistakeCount:
    min: 2
```

### 8.2 向いている用途

```text
向いている用途
├─ 本番前の弱点総復習
├─ 親が必ず入れたい問題＋自動抽出
├─ 検定直前対策
├─ 定期テスト直前対策
└─ 出題範囲が広い総合演習
```

## 9. 目的別問題集のデータ項目

目的別問題集には、次の情報を持たせる。

```text
purposeSetId:
  目的別問題集ID

title:
  表示名

description:
  説明

purposeType:
  定期テスト / 検定 / 弱点補強 / 先取り / 復習 / 本番形式 / 印刷用

selectionMode:
  explicitIds / query / mixed

questionSetScope:
  対象教材セット

catalogScope:
  対象カタログ

conditions:
  抽出条件

questionIds:
  明示IDリスト

ordering:
  出題順

randomization:
  ランダム出題設定

limit:
  出題数

timer:
  タイマー設定

progressScope:
  進捗管理単位

print:
  印刷設定

status:
  draft / active / archived
```

## 10. 出題順の設計

目的別問題集では、出題順を指定できるようにする。

```text
ordering:
  fixed:
    問題IDの順に出す

  random:
    ランダムに出す

  byDifficulty:
    易しい順、難しい順

  byCatalogOrder:
    カタログ順

  weakFirst:
    苦手問題を先に出す

  recentMistakesFirst:
    最近間違えた問題を先に出す
```

当面は、固定順とランダムの2つで十分である。

## 11. 出題数の設計

```text
limit:
  all:
    全問出題

  count:
    指定問数だけ出題

  perTag:
    タグごとに指定数

  perSection:
    セクションごとに指定数
```

例：

```text
数検5級 1次計算技能：
  30問

スピンバイク用：
  40問

英検3級語彙：
  20問

漢検4級読み：
  50問

定期テスト直前：
  全問
```

## 12. タイマー設定

目的別問題集ごとに、推奨タイマーを持たせる。

```text
timer:
  questionSeconds:
    1問あたりの制限時間

  mode:
    normal / quick / noTimer / exam
```

例：

```text
暗算・語彙：
  questionSeconds: 10

数学文章題：
  questionSeconds: 60

英作文：
  noTimer または exam

本番形式：
  exam
```

## 14. 弱点補強の条件

弱点補強では、学習履歴を使う。

```text
weaknessConditions:
  mistakeCount:
    min: 1

  recentMistake:
    true / false

  correctRate:
    max: 0.7

  correctStreak:
    max: 1

  lastAnsweredBefore:
    日付または日数

  unanswered:
    true / false
```

### 14.1 例

```text
直近ミスだけ：
  recentMistake: true

2回以上ミス：
  mistakeCount.min: 2

正答率70%未満：
  correctRate.max: 0.7

しばらく触っていない：
  lastAnsweredBefore: 14days
```

## 20. 実装段階の分け方

### 20.1 段階1：設計のみ

```text
段階1
├─ この文書を作成
├─ 目的別問題集の概念を確定
├─ 既存アプリは変更しない
└─ 新規問題作成時にタグを意識する
```

### 20.2 段階2：手動の目的別セット

```text
段階2
├─ 明示ID方式だけ試す
├─ 既存形式に近い形で実装
├─ 定期テスト直前セットを作る
└─ 進捗は従来どおり教材単位
```

### 20.3 段階3：抽出条件方式

```text
段階3
├─ 問題メタデータを拡張
├─ タグで抽出する
├─ 検定対策セットを作る
```

```text
  将来的には目的別問題集・問題・学習内容ごとに管理
```
