# Learning Content Model v0.8

## 1. 目的

学校教科・検定試験を共通の学習アプリで扱いながら、それぞれに適した意味的分類と画面表示を実現する。

本設計では、次を分離して管理する。

- 学習内容の意味的分類
- 目的別の問題集
- 問題の出典
- 学校教科における教科書ページ範囲
- アプリが読み込むデータ一式
- 問題ごとの進捗

## 2. 基本原則

### 2.1 問題を再利用可能な最小単位とする

1つの問題を、単元別学習、定期テスト対策、苦手復習など複数の問題集から利用できるようにする。

進捗の原本は `questionId` 単位で保持する。

### 2.2 問題集と学習分類を分離する

学習分類は内容を表し、問題集は利用目的を表す。

例：

- 学習テーマ：花のつくりと種子のでき方
- 問題集：2026年度 第1回定期テスト対策

### 2.3 出典構造と学習構造を分離する

教科書の章・節・項は出典上の構造として管理する。

UIでは章番号などを中心にせず、学習内容が分かる意味的名称を表示する。

## 3. 共通概念

| 概念 | 説明 | 例 |
|---|---|---|
| 学習区分 | 学校教科と検定などの大分類 | 学校教科、検定 |
| 学習対象 | 学習する教科・検定 | 理科、社会、漢検、英検 |
| 学習段階 | 学年または級 | 中学1年、漢検4級、英検3級 |
| 学習分類 | 領域別の上位分類 | 生命、地理、文法 |
| 学習テーマ | UIで選択できる意味的な最小集合 | 花のつくり、時差、現在完了 |
| 問題 | 採点・進捗記録の最小単位 | 一問一答の1問 |
| 問題集 | 目的に応じた問題の集合 | 定期テスト対策、漢検4級学習回 |
| 出典教材 | 問題作成・確認に使う資料 | 教科書、ワーク、プリント、過去問 |

学習分類の階層数と表示名は、学習対象ごとに変更できるものとする。

## 4. 領域別の構造例

### 4.1 学校理科

```text
中学1年 理科
└─ 生命
   └─ いろいろな生物とその共通点
      ├─ 観察の基本と観察器具
      ├─ 花のつくりと種子のでき方
      ├─ 種子植物の分類
      └─ 動物の特徴と分類
```

### 4.2 社会地理

```text
中学1年 社会
└─ 地理
   └─ 世界の姿
      ├─ 世界の国々
      ├─ 緯度・経度
      ├─ 地球儀と世界地図
      └─ 緯度と気温
```

### 4.3 漢検

```text
漢検4級
├─ 学習回・問題集
├─ 出題分野
│  ├─ 読み
│  ├─ 書き取り
│  ├─ 熟語の構成
│  ├─ 漢字の識別
│  ├─ 対義語・類義語
│  ├─ 誤字訂正
│  ├─ 四字熟語
│  ├─ 同音・同訓異字
│  ├─ 部首
│  └─ 送り仮名
└─ 対象漢字・熟語
```

漢検の学習回は、学校教科の単元と同一視しない。

## 5. 学校教科の出典管理

### 5.1 教科書を範囲指定の基準とする

学校の定期テスト範囲は教科書ページで指定されることが多いため、学校教科の問題には対応する教科書ページを設定する。

教科書に答えが直接記載されていない応用・発展問題も、その問題の前提となる教科書ページへ関連付ける。

### 5.2 教科書ページと作成根拠を分ける

- `textbookAnchors`：定期テスト範囲との照合に使う教科書ページ
- `sources`：問題作成や内容確認に使った資料

例：

```javascript
textbookAnchors: [
  {
    materialId: "school_science_jhs1_textbook",
    pageFrom: 22,
    pageTo: 24,
    heading: "花のつくり"
  }
],
sources: [
  {
    materialId: "school_science_jhs1_workbook",
    pageFrom: 12,
    pageTo: 13
  }
]
```

### 5.3 複数教材による網羅性確認

学校教科の問題は、次の順で作成・確認する。

1. 教科書から基本問題を作成する
2. ワーク、自主学習教材、配布プリントを教科書ページへ対応付ける
3. 既存問題で扱っている観点と照合する
4. 抜けている知識・技能・出題形式を補完する
5. 応用・発展問題を追加する
6. 教科書ページと学習テーマの両方から網羅性を確認する
7. 問題案をレビューして採否を決める

## 6. 問題の作成区分

| 区分 | 説明 |
|---|---|
| 教科書基本 | 教科書に直接記載された内容 |
| 教材補完 | ワーク、プリントなどから補った内容 |
| 統合 | 複数資料を組み合わせた内容 |
| 応用 | 教科書範囲の知識を使って考える問題 |
| 発展 | 教科書範囲を起点に内容を広げた問題 |

応用・発展問題についても、関連する教科書ページと学習テーマを必須とする。

## 7. 目的別問題集

問題集は問題を複製せず、問題IDまたは抽出条件によって構成する。

例：

```text
問題集：2026年度 中学1年理科 第1回定期テスト対策

対象範囲：
- 理科教科書 p.20～35
- 学校ワーク p.8～17
- 配布プリント No.1～No.3
```

教科書ページを基準に、教材補完、応用、発展問題も抽出対象にできるものとする。

## 8. 問題に表示する資料

### 8.1 出典資料と表示用資料を分離する

問題作成の根拠となる資料と、学習者へ実際に表示する資料は役割が異なるため、別に管理する。

- `sources`：問題作成や内容確認に使った教科書、ワーク、プリントなど
- `questionMaterials`：問題表示時に見せる画像、表、グラフなど
- `answerMaterials`：解答表示時に見せる資料
- `explanationMaterials`：解説表示時に見せる資料

### 8.2 表示段階ごとに資料を持てるようにする

問題画像を解答表示前に見せる必要がある一方、正解を示す図は解答後まで隠す必要がある。

そのため、問題・解答・解説の表示段階ごとに資料を分ける。

例：

```javascript
questionMaterials: [
  {
    type: "image",
    path: "assets/school/science/jhs1/microscope_question.png",
    alt: "顕微鏡の各部を示した図",
    caption: "図を見て答えなさい。",
    displayOrder: 1
  }
],
answerMaterials: [],
explanationMaterials: [
  {
    type: "image",
    path: "assets/school/science/jhs1/microscope_answer.png",
    alt: "顕微鏡の各部の名称を記した図",
    caption: "顕微鏡の各部の名称",
    displayOrder: 1
  }
]
```

### 8.3 初期対応と将来拡張

初期実装では画像表示から対応する。

データ構造は、将来次の資料形式を追加できる形にする。

- 画像
- 表
- グラフ
- 長文・会話文
- 音声
- 動画

1問に複数の資料を設定できるよう、各表示用資料は配列として保持する。

### 8.4 画像表示の要件

- 問題、解答、解説のどの段階で表示するかを区別できる
- 1問に複数画像を設定できる
- 代替テキストを必須とする
- 必要に応じて説明文を表示できる
- 表示順を指定できる
- スマートフォンで画像が画面からはみ出さない
- 細部を確認する必要がある画像は拡大表示を検討する
- 公開可能な自作画像または利用許諾上問題のない画像を使用する

画像表示の画面実装は、学習内容モデルとフォルダ構造を確定した後の別工程とする。

## 9. 内部ID

| ID | 役割 |
|---|---|
| `questionId` | 問題と問題別進捗を識別する |
| `learningThemeId` | 意味的な学習テーマを識別する |
| `materialId` | 教科書、ワーク、プリントなどを識別する |
| `collectionId` | 目的別問題集を識別する |
| `dataPackageId` | 読み込むデータ一式と保存領域を識別する |

これらの内部名は原則としてUIに表示しない。

## 10. 進捗と保存

- 問題別進捗は `questionId` 単位で保存する
- 学習テーマや出題分野の進捗は問題別進捗から集計する
- 同じ問題を複数の問題集で利用しても進捗は共有する
- `dataPackageId`は静的ファイル運用中の読込・保存領域に使う
- 通常バックアップは全保存領域一括を基本候補とする
- 必要に応じて選択中データだけの個別バックアップも残す

## 11. 既存データの扱い

- 既存の問題IDは進捗互換性のため原則変更しない
- `sourceQuestion`と`topic`は重複しているため統合候補とする
- `subSet`は既存テーマIDから`learningThemeId`への移行元として扱う
- `sourcePage`は表示互換用に残し、数値のページ範囲を追加する
- 現在の`setId`は領域ごとに意味が異なるため再定義する
- `questionSetId`は将来`dataPackageId`と`collectionId`に分離する
